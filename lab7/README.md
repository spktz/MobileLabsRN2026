# Лабораторна робота №7
Цей мобільний додаток є навчальним онлайн-магазином музичних інструментів, розробленим на React Native з використанням Expo Router. Головна особливість проєкту — впровадження системи глобального управління станом через Redux Toolkit. У додатку реалізовано каталог товарів, де можна обрати професійне музичне обладнання, таке як гітари Fender, синтезатори Roland чи барабани Yamaha, та додати їх до кошика.

Для запуску проєкту необхідно завантажити репозиторій та виконати команду npm install, щоб встановити всі бібліотеки для роботи з Redux, AsyncStorage та навігацією. Після цього сервер запускається командою npx expo start. Оскільки додаток використовує систему маршрутизації Expo Router, точка входу налаштована через expo-router/entry. Для перевірки роботи на смартфоні достатньо відсканувати QR-код через Expo Go, а для запуску на комп'ютері натиснути клавішу "a", щоб відкрити Android-емулятор.

Під час виконання роботи було реалізовано складну логіку кошика, де користувач може не лише додавати інструменти, а й змінювати їхню кількість або видаляти зайві позиції, при цьому загальна сума замовлення перераховується автоматично. Також створено систему оформлення замовлення з формою ПІБ та контактними даними, яка після валідації зберігає покупку в окрему історію замовлень. Завдяки використанню бібліотеки redux-persist усі дані, включаючи кошик та історію, зберігаються в пам'яті телефону через AsyncStorage, тому вони не зникають навіть після повного перезапуску додатка.

Що таке глобальний стан у React Native?
Це така спільна пам’ять для всього додатка. Якщо звичайний useState працює тільки всередині одного екрана, то глобальний стан дозволяє діставати дані (наприклад, список товарів у кошику) з будь-якого місця в програмі без зайвої передачі параметрів.

Для чого використовується Redux Toolkit?
Це сучасний набір інструментів, який робить роботу з Redux простішою. Він дозволяє писати менше коду, автоматично налаштовує сховище і допомагає уникнути помилок, які часто траплялися в старому Redux.

Яке призначення createSlice?
Це функція, яка дозволяє згрупувати логіку для певної частини додатка в одному місці. В «слайсі» ми одразу пишемо і початковий стан (наприклад, порожній кошик), і всі дії, які можна з ним робити (додати товар, видалити або змінити кількість).

Для чого використовується configureStore?
Він створює головний "центр керування" або сховище (Store). Ця функція збирає всі наші слайси докупи, підключає до них різні корисні додатки (як-от логування чи збереження даних) і робить стан доступним для всього додатка.

Для чого використовується redux-persist?
Він потрібен, щоб дані не зникали після того, як ми повністю закрили додаток або перезавантажили телефон. Він автоматично бере наш стан з Redux і зберігає його в постійну пам'ять пристрою (AsyncStorage).
